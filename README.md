## UI components & copy-paste blocks

Copy production-ready UI into your React/Vite/Next.js projects (often Tailwind + shadcn-style).

### Copy/paste-first (blocks and sections)

- [Shadcn Blocks](https://shadcnblocks.com) — Large library of shadcn/ui + Tailwind blocks (landing pages, pricing, dashboards).
- [Magic UI](https://magicui.design) — Polished animated UI components (bento grids, effects, interaction-heavy sections).
- [Cult UI](https://cult-ui.com) — Niche animated components built to fit the shadcn/ui ecosystem (great for “AI app” layouts).
- [Uiverse](https://uiverse.io) — Huge open-source library of small UI elements (buttons, toggles, inputs, cards).

### More free component libraries (copy + customize)

- [21st.dev](https://21st.dev) — Community-driven React + TypeScript + Tailwind components. Copy, paste, customize.
- [Aceternity UI](https://ui.aceternity.com) — Free React components + Tailwind blocks (copy/paste friendly with elegant animations).
- [HyperUI](https://hyperui.dev) — Tailwind + React blocks/components library optimized for SaaS, marketing, and e-commerce.
- [Float UI](https://floatui.com) — Free Tailwind components (good for marketing + app UI, compatible with Vue, Svelte, and React).

### Installable (when you *want* a framework/plugin)

- [DaisyUI](https://daisyui.com) — Tailwind CSS component plugin (fast theming + common semantic components).
- [Flowbite](https://flowbite.com) — Tailwind component library + docs for interactive components and UI layouts.

## Animations & micro-interactions

- [Animista](https://animista.net) — Generate CSS animations; tweak easing/duration and copy keyframes instantly.
- [Loading.io](https://loading.io) — Generate CSS/SVG for “SVG draw” and spinner animations (great for icons/logos).
- [CSS Gradient](https://cssgradient.io) — Gradient playground; copy clean CSS for linear/radial color transitions.

## Backgrounds, patterns & SVG polish

Fast “premium look” without heavy images.

- [PatternCraft](https://fountn.design) — Copy-ready CSS/Tailwind patterns & subtle geometric grids.
- [Super Designer](https://superdesigner.co) — Free generators for mesh gradients, blobs, backgrounds, palettes, and OG images.
- [Haikei](https://haikei.app) — Generate SVG assets (waves, blobs, gradients) and export custom SVG/PNG layout elements.
- [SVG Backgrounds](https://www.svgbackgrounds.com) — SVG background generator with many styles, presets, and scaling controls.
- [Hero Patterns](https://heropatterns.com) — Repeatable SVG patterns with live color adjustments and CSS snippets.
- [BGJar](https://bgjar.com) — Free SVG generator for abstract backgrounds, tech textures, and geometric maps.

## Browser extension starters & tooling

### Modern extension frameworks / build tools

- [WXT](https://wxt.dev) — Next-gen web extension framework supporting multi-browser targets and great developer experience.
- [CRXJS](https://crxjs.codes) — Build modern browser extensions with native Vite and HMR-style workflows.
- [vite-plugin-web-extension](https://github.com/aklinker1/vite-plugin-web-extension) — Lightweight Vite plugin for building cross-browser web extensions cleanly.
- [Plasmo](https://www.plasmo.com) — Robust extension framework with out-of-the-box support for Tailwind, React, and automated store deployments.

### Cross-browser + packaging helpers

- [Webextension Polyfill](https://github.com/mozilla/webextension-polyfill) — Promise-based `browser.*` API compatibility layer for cross-browser extensions.
- [Web-Ext (Mozilla)](https://github.com/mozilla/web-ext) — Mozilla command line interface tool to run, build, test, and sign extensions easily.
- [MDN Web Extensions Docs](https://developer.mozilla.org/en-US/docs/Mozilla/Add-ons/WebExtensions) — The best all-around reference and standard documentation for Manifest V3 design.

### Quick notes (practical)
> 💡 **Framework Tip:** Prefer a modern tool like **WXT**, **CRXJS**, or **Plasmo** if you want fast hot-reloading (HMR), simpler Manifest V3 setups, and pain-free multi-browser output builds. 
> 
> 🔀 **Cross-Browser Dev:** Drop the `webextension-polyfill` into your source directory early to abstract standard asynchronous API discrepancies between Chromium and Firefox.

## Fonts & typography

### Font sources & directories

- [Google Fonts](https://fonts.google.com) — Massive open directory of free-tier web fonts.
- [Fontsource](https://fontsource.org) — Self-host open fonts via specific npm packages (great for performance, privacy, and control).
- [Fontshare](https://www.fontshare.com) — Curated directory of professional, open-source display and text fonts by the Indian Type Foundry.
- [DaFont](https://www.dafont.com) — "Every Free Font" style directory for rapid brainstorming (ensure to filter and review licenses per asset).
- [Velvetyne](https://velvetyne.fr) — Free and open-source custom font collective pushing experimental typographic design styles.
- [Open Foundry](https://open-foundry.com) — Open-source experimental type library cataloged neatly for digital layouts.

## Design, wireframing & diagrams

- [Penpot](https://penpot.app) — Open-source web design + prototyping tool featuring full native CSS layout and grid support.
- [Excalidraw](https://excalidraw.com) — Open-source virtual whiteboard for sketchy flows, architecture wireframes, and fast system diagrams.

## Optimization & performance

- [Squoosh](https://squoosh.app) — In-browser high-efficiency image compression engine supporting advanced WebP/AVIF workflows.
